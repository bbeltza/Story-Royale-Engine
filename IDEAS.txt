- Some game engine hit ideas (designed for more expertise discussion):
    - SRE_HINT_ARG_HANDLER & SRE_HINT_ARG_HANDLERS: ARG_HANDLER is a pointer to a function, taking the signature of `main` (although I'm not sure about the return type) that is used to handle command line arguments.
                                                      so the game can now handle arguments, the function is run on the main thread before any initialization is done so be careful!
                                                        this might also be used to change hints at runtime (ex: the currently existing renderer driver hint whether you decide on the command line to specify a special argument)
                                                      if ARG_HANDLERS is present, then ARG_HANDLER will be taken as an array of function pointers with that same signature, ARG_HANDLERS being the amount of argument handlers present in that array.
                                                      if there's no ARG_HANDLERS, then it is assumed for ARG_HANDLER to have only 1 function, so you could either declare it as an array of a single function or just a function pointer, it doesn't really matter.
    - SRE_HINT_WINDOW_SIZE: It's an array of two integers (so you can declare it as a vec2i, just a C array or an std::array<int>, or heck even a long long if you're careful about endianness!). The array defines the dimensions of the window on startup.
                                Obviously, you know, the first element of the array will contain the width, and the second element the height. Very simple. More of these hints will be also there.
                                If there's no SRE_HINT_WINDOW_SIZE, then the engine will create a window with a default size of 800x600 pixels.
    - SRE_HINT_WINDOW_POS: Same as SRE_HINT_WINDOW_POS, but sets up the main window's position at the start of the game (I'm saying "main window" because these hints will be supported after the new window API where multiple windows are supported).
                                The window will be positioned at the coordinates given by SRE_HINT_WINDOW_POS with the top-left corner as the origin.
                                Although not recommended for the hint. Setting up values lower than 0 may place the window inside other monitors if any, since their desktop-space origins may be lower than 0 depending on where it is placed relative to the main monitor.
                                If this flag is not set, then the window will be created with SDL_WINDOWPOS_CENTERED as the x and y coordinates. Which will often place the window at the center of the main screen.
    - SRE_HINT_SRGB: A boolean value indicating to the render driver whether to use an sRGB color or not. It is by default set to `false`.
